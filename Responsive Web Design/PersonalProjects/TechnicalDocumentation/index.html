Responsive web design

Responsive web-design is an approach to web design that allows us to create functional websites for all kind of devices. Before this became the standard way of making websites, we referred to it as mobile-friendly design or mobile web development, but the goal is exacty the same: making sure our websites will work exactly the same way, on our mobile and on our laptop.

Nowadays, with plenty of different sizes devices, we need to design websites that properly work on every kind of common screen, plus many unkowns, so it is obvious that we need a more generalized and standard way to do this.

Setting the viewport

The viewport meta tag defines for mobile browsers that the width of the viewport should be set to the device width.

          
            <meta name="viewport">

This was not working though, as it would show a zoomed-out version of the desktop layout and users would need to zoom in, but it looked untidy. This is why we use content, as we override a mobile device's default, with the actual width of the device. For this reason, bth these tags should always be included in the head of the document. 

          
            <meta name="viewport" content="width=device-width,initial-scale=1">
Responsive Images
  • Width property

    • Using the width property and setting it to 100% will render a responsive image and scale it up and down.

    <img src="img_girl.jpg" style="width:100%;">
  • Max-width property

    • A better solution is using max-width, set to 100%. In this case the image will scale down if needed, but it will never scale up to a larger size than its original one.

    <img src="img_girl.jpg" style="max-width:100%;height:auto;">
  • Picture/image tag

    • The picture tag allows to define different images for different browser windows sizes. 

    <img src="img_girl.jpg" style="max-width:100%;height:auto;">

    It is infact an element that contains several source elements, providing different sources for the browser to choose from, followed by the img element.

                    
                <picture>
                  <source media="(max-width: 799px)" srcset="elva-480w-close-portrait.jpg" />
                  <source media="(min-width: 800px)" srcset="elva-800w.jpg" />
                  <img src="elva-800w.jpg" alt="chris standing up holding his daughter elva" />
                </picture>

    The source elements include a media attribute that contains a condition: if the first one returns true, the first source element's image will be displayed, otherwise it will be the second one.

    The srcset attributes contain the path to the image to display.

    In any case, after the picture element, an img element must be provided, together with src and alt, to make sure the image appears whenever none of the default cases apply.

Responsive typography

The text size can also be set with a vw unit: the viewport width. This way the text will take the size of the browser window.

<h1 style="font-size:10vw">hello world</h1>

Viewport is the browser window size. 1vw = 1% of viewport width.

Media queries

Media queries allow us to run tests to check whether the user's screen is greater than a certain width or resolution, and apply CSS in a selective way, so to style the page according to the user's needs.

@media screen and (min-width: 80rem) {
            .container {
              margin: 1em 2em;
            }
          }

In this specific case, the media query has two conditions and will only apply the CSS styling if the web page is displayed as screen media (so not a printed document) and if the viewport is at least 80rem wide. Multiple media queries can be added within a stylesheet and the points at which the query is introduced, are known as breakpoints.

Responsive Layout CSS technologies
  • Multicol

    • Using multicol, we can specify a column-count that will indicate the maximum number of columns our content will be split into. Thw size of the columns will change according to the screen size.

    .container {
              column-count: 3;
            }

    When specifying a column-width we specify a minimum width, so as many columns of the chosen widths fit into the container will be created, adapting to the available space.

    .container {
              column-width: 10em;
            }

    We can also use columns as shorthand property

  • Flexbox

    • When we use flexbox, the items will shrink or grow and the space will be distributed according to the space in the main container.

                  .container {
                  display: flex;
                }
                
                .item {
                  flex: 1;
                }

    In this example, every item will take an equal amount of space in the flex container.

  • Grids

    • With grids, we have a unit measure called fr which determines the distribution of the available space across grid tracks. In the case below for example, the container will have three column tracks, each one taking part of the available space in the main container. 

    .container {
              display: grid;
              grid-template-columns: 1fr 1fr 1fr;
            }
